KEY FEATURES EXPLAINED
================================================================================

FEATURE 1: HASH COMPUTATION
────────────────────────────────────────────────────────────────────────────

What: Computes SHA256 hash for every file
Why: 
  • Verify file integrity
  • Compare against known-good databases (NSRL)
  • Identify modified files
  • Legal evidence documentation

Example Output:
  malware.exe: 80d683557785fe07 (SHA256 truncated to first 16 chars)
  
Real Mode: Reads actual file data and computes true hash
Demo Mode: Generates symbolic hash based on filename+size

Usage:
  # In JSON report:
  "file_hashes": {
    "256_malware.exe": "80d683557785fe07",
    ...
  }

FEATURE 2: TIMELINE ANALYSIS
────────────────────────────────────────────────────────────────────────────

What: Creates chronological reconstruction of file modifications
Why:
  • Identify attack timeline
  • Detect suspicious clustering (multiple edits at odd hours)
  • Correlate with event logs
  • Establish actor activity periods

Example Output:
  [2026-02-21 14:20:00] MODIFICATION: /pagefile.sys (2.1 GB)
  [2026-02-21 04:30:00] MODIFICATION: /hiberfil.sys (4.3 GB)
  [2025-10-15 22:45:00] MODIFICATION: /Users/Admin/Downloads/malware.exe (2 MB)

Analysis:
  - Recent system file activity (02:20 and 04:30 = typical off-hours)
  - Malware appeared Oct 15, 2025 at 22:45 (10:45 PM)
  - Suggests after-hours installation or automated schedule

FEATURE 3: FILE EXTENSION DISTRIBUTION
────────────────────────────────────────────────────────────────────────────

What: Statistical breakdown of all file types
Why:
  • Identify unusual file patterns
  • Spot suspicious installations
  • Detect data exfiltration attempts
  • Forensic baseline comparison

Example Output:
  .exe: 2 files (15.38%)      ← Executables
  .sys: 2 files (15.38%)      ← System files
  .dll: 2 files (15.38%)      ← Libraries
  .ps1: 1 file (7.69%)        ← PowerShell scripts
  .bat: 1 file (7.69%)        ← Batch scripts

Red Flags:
  - .exe, .bat, .ps1, .dll in user directories
  - .scr, .vbs, .js files = script-based malware
  - Unusual extensions = suspicious

FEATURE 4: SUSPICIOUS FILE DETECTION
────────────────────────────────────────────────────────────────────────────

What: Highlights 16 file extensions associated with malware/attacks
Suspicious Extensions:
  .exe, .bat, .ps1, .dll, .scr, .vbs, .js, .com, .msi, .sys,
  .pif, .cmd, .jar, .sh

Example Detection:
  ✓ malware.exe (Downloads/) ← Executable in user folder = SUSPICIOUS
  ✓ script.ps1 (Documents/) ← PowerShell = Remote execution capability
  ✓ batch_file.bat (Startup/) ← Runs at boot = Persistence mechanism
  ✓ ransomware.dll (System32/drivers/) ← DLL in drivers = High privilege

FEATURE 5: ENCRYPTION DETECTION
────────────────────────────────────────────────────────────────────────────

What: Identifies encrypted volumes and protected archives
Detects:
  • TrueCrypt/VeraCrypt containers
  • BitLocker encrypted volumes
  • LUKS encrypted drives
  • PGP encrypted files
  • Compressed/encrypted archives (.7z, .zip, .rar, .iso)

Example Detection:
  encrypted.img (10.7 GB) in Documents/
  
  Forensic Significance:
    - Large encrypted file suggests data hiding
    - Hidden from normal analysis
    - Requires password recovery or key material
    - Could contain evidence or stolen data

FEATURE 6: NETWORK ARTIFACTS
────────────────────────────────────────────────────────────────────────────

What: Identifies network-related files and configurations
Detected Artifacts:
  • hosts file (/Windows/System32/drivers/etc/hosts)
  • Browser history and cache
  • Email configuration
  • VPN/proxy settings
  • DNS configuration
  • Network connection logs

Example:
  Found: hosts file
  
  Significance:
    - Modified 2025-11-10 (recent change)
    - Could contain DNS redirects for malware C&C
    - Check for suspicious IP mappings

HOW TO USE WITH REAL DISK IMAGES
================================================================================

Step 1: Install Dependencies
────────────────────────────────────────────────────────────────────────────

# Option A: RAW images only
pip install pytsk3

# Option B: RAW + .E01 images
pip install pytsk3 pyewf

# Option C: Install build tools first (if errors)
# Download: https://visualstudio.microsoft.com/downloads/
# Select: Visual Studio Build Tools 2022 → Desktop development with C++

Step 2: Update Configuration
────────────────────────────────────────────────────────────────────────────

Edit forensic_pipeline_advanced.py:

  IMAGE_PATH = r"C:\path\to\image.raw"   # or image.E01
  USE_DEMO = False  # Change from True to False
  
  JSON_OUTPUT = r"D:\path\report.json"
  SUMMARY_OUTPUT = r"D:\path\summary.txt"
  TIMELINE_OUTPUT = r"D:\path\timeline.txt"
  EXTENSION_OUTPUT = r"D:\path\extensions.txt"

Step 3: Run Analysis
────────────────────────────────────────────────────────────────────────────

python forensic_pipeline_advanced.py

Monitor console output:
  [*] Opening disk image...
  [+] Partition detection...
  [+] File scanning...
  [*] Computing hashes...
  [+] All reports generated!

Step 4: Review Reports
────────────────────────────────────────────────────────────────────────────

1. Open forensic_summary_advanced.txt for quick overview
2. Check forensic_timeline.txt for activity timeline
3. Review forensic_extensions.txt for file distribution
4. Use forensic_report_advanced.json for detailed analysis

TROUBLESHOOTING
================================================================================

Problem: "ModuleNotFoundError: No module named 'pytsk3'"
Solution: pip install pytsk3

Problem: "Microsoft Visual C++ 14.0 or greater is required"
Solution: Install Visual Studio Build Tools 2022
  Download: https://visualstudio.microsoft.com/downloads/
  Select: Desktop development with C++

Problem: "No partitions detected"
Solution: This is normal - script uses entire image as single partition

Problem: "Error opening filesystem"
Solution: 
  - Verify image file is not corrupted
  - Try with different filesystem type
  - Check file permissions

Problem: "LLM returned status 404"
Solution: Local LLM not running. Start Ollama:
  ollama serve

Problem: LLM takes too long
Solution: Use smaller model
  ollama pull mistral (7B instead of 13B+)

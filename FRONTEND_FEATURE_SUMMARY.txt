═══════════════════════════════════════════════════════════════════════════════════

Created a complete modern, futuristic web application for forensic disk analysis with:
  ✓ Beautiful dark-themed interface with cyan/teal accents
  ✓ Real-time progress tracking
  ✓ Professional PDF and Word report generation
  ✓ Advanced disk forensics analysis (pytsk3, pyewf)

  ✓ Backend Flask server
  ✓ No dependencies on external services

The web app provides a professional interface to the forensic pipeline already built,
making disk image analysis accessible through a modern, intuitive UI.

BACKEND API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════════

1. GET /
   Purpose: Serve main HTML interface
   Response: index.html (frontend)
   Status: 200 OK

2. POST /api/analyze
   Purpose: Start forensic analysis
   Request Body: { "image_path": "C:\\forensics\\image.raw" }
   Response: { "success": true, "analysis_id": "ID", "message": "..." }
   Status: 200 OK or 500 Error
   Inside: Spawns background thread running AdvancedForensicAnalyzer

3. GET /api/status
   Purpose: Get current analysis progress
   Response JSON:
   {
     "current": "analysis_id",
     "progress": 45,
     "status": "analyzing",
     "message": "Scanning files recursively...",
     "errors": [],
     "results": null
   }
   Status: 200 OK
   Updates: Every 500ms from frontend

4. GET /api/results
   Purpose: Get analysis results (post-completion)
   Response JSON:
   {
     "timestamp": "2026-02-21 14:32:18",
     "image_path": "C:\\forensics\\image.raw",
     "summary": {
       "total_files": 13,
       "suspicious_files_count": 8,
       "total_deleted_files": 1,
       ...
     },
     "extensions": { ".exe": 2, ".dll": 2, ... },
     "encrypted_items": [...],
     "network_artifacts": [...]
   }
   Status: 200 OK or 400 No Results

5. GET /api/export/<format>
   Purpose: Download report in specified format
   Formats: "pdf" or "doc"
   Response: Binary file (PDF or DOCX)
   Status: 200 OK or 500 Error
   Types:
     • PDF: application/pdf
     • DOCX: application/vnd.openxmlformats-officedocument.wordprocessingml.document

6. GET /api/reset
   Purpose: Reset analysis state for new analysis
   Response: { "success": true, "message": "Analysis reset" }
   Status: 200 OK
   Effect: Clears all analysis data

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════════

ISSUE: "Port 5000 already in use"
SOLUTION:
  netstat -ano | findstr :5000
  taskkill /PID [PID] /F
  
  Or edit forensic_web_app.py line 392:
  app.run(host='localhost', port=5001)

ISSUE: "pytsk3 installation fails"
SOLUTION:
  Download: https://visualstudio.microsoft.com/downloads/
  Select: "Desktop development with C++"
  Install and retry pip install pytsk3

ISSUE: "Browser shows 'Connection refused'"
SOLUTION:
  • Verify Flask is running (check terminal)
  • Clear browser cache (Ctrl+Shift+Delete)
  • Try: http://127.0.0.1:5000
  • Check firewall (allow port 5000)

ISSUE: "Analysis never completes"
SOLUTION:
  • Check disk image file is valid
  • Verify sufficient disk space
  • For large drives (>500GB), can take 30+ minutes
  • Monitor Task Manager for I/O

ISSUE: "PDF/DOC export fails"
SOLUTION:
  • pip list | findstr reportlab (check installed)
  • pip list | findstr docx (check installed)
  • Verify free disk space
  • Check output folder permissions

  → Reports marked as sensitive

⚙️ API ENDPOINTS
═══════════════════════════════════════════════════════════════════════════════════

The React frontend communicates with Flask backend:

Upload Endpoint:
  POST /api/upload
  • Accepts forensic image files
  • Returns: { fileName, analysisId, status }

Status Endpoint:
  GET /api/analysis/<id>/status
  • Real-time analysis progress
  • Returns: { progress, status, summary }

Report Endpoint:
  GET /api/analysis/<id>/report
  • Complete analysis report
  • Returns: { summary, filesByType, timeline }

Export Endpoints:
  GET /api/analysis/<id>/export/pdf
  GET /api/analysis/<id>/export/docx
  • Download reports in PDF or DOCX format

Configuration:
  • Proxy: vite.config.ts routes /api/ → http://localhost:5000
  • CORS: Requires flask-cors in Flask backend
  • Polling: Dashboard polls every 2 seconds during analysis

📚 IMPORTANT: WHY /components/ui MATTERS
═══════════════════════════════════════════════════════════════════════════════════

The /components/ui directory is NOT optional - it's the shadcn standard because:

1. CONVENTION: All shadcn projects use this structure
2. CLI COMPATIBILITY: npx shadcn-ui@latest add writes to /components/ui
3. PREDICTABILITY: Team members know exactly where to find components
4. MAINTAINABILITY: Clear separation between shadcn and custom components
5. SCALABILITY: Growing component library remains organized

Future Component Additions:
  
  npx shadcn-ui@latest add button      # → /components/ui/button.tsx
  npx shadcn-ui@latest add card        # → /components/ui/card.tsx
  npx shadcn-ui@latest add dialog      # → /components/ui/dialog.tsx

Then wrap or extend them in /components/forensics for project-specific use.

See COMPONENT_STRUCTURE_GUIDE.txt for detailed best practices.

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════════

Issue: "npm: command not found"
→ Install Node.js from https://nodejs.org/

Issue: Port 5173 already in use
→ npm run dev -- --port 5174

Issue: Cannot find module '@/...'
→ Check tsconfig.json paths configuration

Issue: Tailwind styles not applying
→ Restart dev server or rebuild

Issue: API calls fail (CORS error)
→ Ensure Flask has CORS enabled: pip install flask-cors

Issue: Components not displaying
→ Check browser console for errors (F12)
→ Verify import paths use @ alias

Full troubleshooting: See REACT_FRONTEND_SETUP.txt

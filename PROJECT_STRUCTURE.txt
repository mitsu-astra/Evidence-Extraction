DATA FLOW
═════════════════════════════════════════════════════════════════════════════════════

1. USER OPENS APP
   └─ http://localhost:5173
   └─ ForensicAnalysisHero renders

2. USER CLICKS "BEGIN ANALYSIS"
   └─ App state changes
   └─ Shows ForensicFileUpload

3. USER UPLOADS FILE
   └─ ForensicFileUpload uploads to /api/upload
   └─ Backend receives file
   └─ Analysis starts

4. ANALYSIS STARTS
   └─ ForensicAnalysisDashboard receives event
   └─ Creates analysis record
   └─ Starts polling /api/analysis/<id>/status

5. ANALYSIS RUNNING
   └─ Dashboard shows progress bar
   └─ Polls every 2 seconds
   └─ Updates progress percentage

6. ANALYSIS COMPLETE
   └─ Status changes to "completed"
   └─ Dashboard fetches final report
   └─ Shows summary and statistics

7. USER EXPORTS REPORT
   └─ Calls /api/analysis/<id>/export/pdf or docx
   └─ File downloads to computer


API ENDPOINTS
═════════════════════════════════════════════════════════════════════════════════════

From React to Flask:

POST /api/upload
    ├─ Request: FormData with 'file'
    └─ Response: { fileName, analysisId, status }

GET /api/analysis/<id>/status
    ├─ Response: { progress, status, summary }
    └─ Called every 2 seconds during analysis

GET /api/analysis/<id>/report
    ├─ Response: { summary, filesByType, timeline }
    └─ Called when analysis completes

GET /api/analysis/<id>/export/pdf
    ├─ Response: PDF binary file
    └─ User clicks "Export as PDF"

GET /api/analysis/<id>/export/docx
    ├─ Response: DOCX binary file
    └─ User clicks "Export as DOCX"

HOW TO USE
═════════════════════════════════════════════════════════════════════════════════════

First Time:
  1. npm install
  2. npm run dev
  3. Open http://localhost:5173

Development:
  1. npm run dev (keeps running)
  2. Edit files in src/
  3. Changes auto-reload in browser

Adding Features:
  1. Create files in src/components/forensics/
  2. Import in App.tsx or other components
  3. Use Tailwind classes for styling

Building:
  1. npm run build
  2. Output in dist/ folder
  3. Deploy dist/ folder

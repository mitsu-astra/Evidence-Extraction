# Response Source Tracking - `generated_by` Field

## Overview
All RAG chatbot responses now include a `generated_by` field that indicates the source of the response.

## Response Sources

### 1. **`"generated_by": "llama3"`**
- The response was **generated by Llama 3** AI model via Ollama
- Indicates high-quality, natural language forensic analysis
- Used when:
  - User asks a forensic-related question
  - Ollama server is running and responsive
  - Retrieved forensic context is available

**Example:**
```json
{
  "answer": "According to the forensic report, the following files were flagged as suspicious: malware.exe, script.ps1, batch_file.bat...",
  "generated_by": "llama3",
  "context": [{...}]
}
```

---

### 2. **`"generated_by": "fallback"`**
- The response was **generated by rule-based Python code** (backup system)
- No LLM used (either Ollama is down, or `use_llm=false`)
- More structured/templated responses, still accurate but less conversational
- Used when:
  - Ollama connection fails or times out
  - User sends `use_llm=false` in the request
  - Query syntax matches predefined fallback patterns

**Example:**
```json
{
  "answer": "🔴 **Suspicious files detected in the report:**\n...",
  "generated_by": "fallback",
  "context": [{...}]
}
```

---

### 3. **`"generated_by": "chat"`**
- The response was **generated from canned chat responses** (small talk)
- Friendly conversational replies to greetings, small talk, etc.
- Fuzzy-matched (handles typos like "hlo" → "hello")
- Used when:
  - User sends greetings ("hi", "hello", "hey", etc.)
  - User asks about capabilities ("what can you do?")
  - User thanks the chatbot

**Example:**
```json
{
  "answer": "👋 Hi there! I'm the Forensic RAG Assistant. You can ask me things like: What suspicious files were found?...",
  "generated_by": "chat",
  "context": []
}
```

---

## API Usage

### Request
```bash
POST /api/rag/chat
{
  "query": "What suspicious files were found?",
  "use_llm": true,
  "top_k": 5
}
```

### Response
```json
{
  "success": true,
  "answer": "According to the forensic report...",
  "generated_by": "llama3",
  "context": [{...}],
  "collection": "forensic_test_analysis_001",
  "rejected": false
}
```

## Frontend Integration

### Display Response Source
```typescript
// React component example
const ChatResponse = ({ answer, generated_by }) => {
  const sourceIcons = {
    'llama3': '🤖',   // AI-generated
    'fallback': '📄',  // Rule-based
    'chat': '👋'       // Small talk
  };

  const sourceLabels = {
    'llama3': 'Generated by Llama 3 AI',
    'fallback': 'Generated by Backup System',
    'chat': 'Quick Response'
  };

  return (
    <div className="response">
      <div className="source-badge">
        {sourceIcons[generated_by]} {sourceLabels[generated_by]}
      </div>
      <div className="answer">{answer}</div>
    </div>
  );
};
```

## Debugging & Troubleshooting

### If you see `"generated_by": "fallback"` for forensic queries:
- **Check Ollama status**: `curl http://localhost:11434/api/tags`
- **Check Llama3 model**: `ollama list` (should show `llama3:latest`)
- **Check logs**: Look for connection errors in Flask app

### If you see `"generated_by": "llama3"` but answers are poor quality:
- Context retrieval might be retrieving irrelevant chunks
- Try adjusting `top_k` parameter (more/fewer context chunks)
- Consider adding more diverse forensic data to vectorize

### If you see `"generated_by": "chat"`:
- User is asking a greeting or small-talk question
- Normal behavior - try asking a forensic question to get AI/fallback responses
